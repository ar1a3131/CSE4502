    tp = 0
    tn = 0
    fp = 0
    fn = 0    
    for i in range(len(y_true)):
        if y_true[i] == 1 and y_predicted[i] == 1:
            tp += 1
        # YOUR CODE
        elif y_true[i] == 0 and y_predicted[i] == 0:
            tn += 1
        elif y_true[i] == 1 and y_predicted[i] == 0:
            fn += 1
        elif y_true[i] == 0 and y_predicted[i] == 1:
            fp += 1

    # Calculate Sensitivity and Specificity
    sensitivity = tp / (tp + fn) if (tp + fn) > 0 else 0
    specificity = tn / (tn + fp) if (tn + fp) > 0 else 0

    # Calculate Balanced Accuracy
    balanced_acc = (sensitivity + specificity) / 2

Run the following code and answer the following questions:

(a) How does accuracy change with changing the tree maximum depth (line 17)?

The maximum depth (max_depth) of the tree affects the complexity of the model. If max_depth is set too low, the decision tree may be too simple to capture the underlying patterns of the data, leading to underfitting, which typically results in a lower accuracy.

On the other hand, setting max_depth too high can lead to a very complex model that captures noise in the training data rather than the actual signal, leading to overfitting. When overfitted, the model may show very high accuracy on the training data but poor accuracy on new and unseen data.

The ideal value for max_depth would balance model complexity and generalization by being high enough to capture relevant patterns but not so high that the model learns the "noise" in the training data.

(b) What does the 'min_samples_leaf' parameter represent in a DecisionTreeClassifier according to the sklearn documentation, and how is it connected to the concepts of overfitting or underfitting?

The min_samples_leaf parameter sets the minimum number of samples that are allowed in leaf nodes of the tree. Setting this parameter can help in controlling overfitting:

Higher values of min_samples_leaf can lead to simpler, more generalized models because forcing each leaf to have more samples means the tree is less able to learn the fine, possibly noisy, specifics of the training data, therefore reducing overfitting.

Lower values of min_samples_leaf allow the tree to make splits that contain fewer samples, which can create a more complex model capable of capturing more information about the data, including noise, potentially leading to overfitting. If set too low, such as 1, the model will likely fit perfectly to the training data, capturing noise and anomalies, and thus not perform well on unseen data.

(c) List three additional parameters of a DecisionTreeClassifier as per the sklearn documentation.

criterion: This parameter defines the function to measure the quality of a split. Supported criteria are "gini" for the Gini impurity and "entropy" for the information gain.

splitter: This parameter defines the strategy used to choose the split at each node. Supported strategies are "best" to choose the best split and "random" to choose the best random split.

max_features: This parameter limits the number of features to consider when looking for the best split. It can be an integer (to specify the exact number), a float (to specify a percentage of the total), or one of "auto", "sqrt", or "log2", which are shortcuts for these options based on the number of features.

from sklearn import datasets
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.tree import plot_tree
from sklearn.tree import DecisionTreeClassifier
#added
import sklearn.metrics as metrics
import matplotlib.pyplot as plt


iris = datasets.load_iris()
X = iris.data[:, [2, 3]]
y = iris.target


X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=1, stratify=y)

tree = DecisionTreeClassifier(criterion='entropy', 
                              max_depth=10, 
                              random_state=1)
tree.fit(X_train, y_train)

y_pred = tree.predict(X_test)
test_accuracy = metrics.accuracy_score(y_test, y_pred)
print("Test accuracy of decision tree classifier on Iris dataset: "+str(test_accuracy))

plt.figure(figsize=(10, 7))

plot_tree(tree, 
          filled=True, 
          rounded=True,
          class_names=['Setosa',
